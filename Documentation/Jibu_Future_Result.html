Waits until the Future has finished execution, and returns the result.

C++
public: T Result();
C#
public T Result();
Visual Basic
Public Function Result() As T
Remarks

Waits until the Future has finished execution, and returns the result. 

If the Start method hasn't been invoked, then the Future will be executed immediately in this thread, bypassing the Jibu scheduler. 

If the Future has been scheduled and is running, then Result waits until it has finished execution else if the Future is already done, Result returns immediately. 

Any uncaught exception thrown by the executed task will be rethrown by Result, encapsulated in a CancelException. If the Future is cancelled a CancelException is thrown.

Exceptions
Exceptions 
Description 
Jibu.CancelException 
Thrown if the Future has been cancelled, or if user code has generated an exception. In the latter case the InnerException property will be set. 
Future Example
Copy Code
using System;
using Jibu;

namespace FutureExample
{
    // Simple task that can calculate Fibonacci numbers
    class FibonacciTask : Future<int>
    {
        int n;

        public FibonacciTask(int n)
        {
            this.n = n;    
        }

        private static int Fibonacci(int n)
        {
            if(n <= 2)
                return 1;
            else
                return Fibonacci(n - 2) + Fibonacci(n - 1);            
        }

        public override int Run()
        {
            return Fibonacci(n);
        }

    }

    class FutureExample
    {
        static void Main(string[] args)
        {
            int n = 40;

            // Start the asynchronous task
            Future<int> fibonacciTask = new FibonacciTask(n).Start();

            // You can do some work here....

            Console.WriteLine("Waiting for the Future to finish...");

            // ... and then get the result later
            Console.WriteLine("Fibonacci({0}) = {1} ", n, fibonacciTask.Result());


            // This task throws an exception...
            Future<int> exceptionTask = new DelegateFuture<int>(delegate 
            { 
                throw new Exception("Goodbye world!"); 
            }).Start();

            try
            {
                int res = exceptionTask.Result();
            }
            catch (CancelException e)
            {
                // ... but it isn't propagated out until 
                // we call the Result method.
                // Result throws the caught exception 
                // wrapped in a Jibu.CancelException.
                // The original exception can be 
                // accessed through the InnerException
                // property.
                if(e.InnerException != null)
                    Console.WriteLine("Success! We caught an exception: " + e.InnerException.Message);
            }

        }
    }
}
